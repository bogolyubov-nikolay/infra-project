# Задание 9: CI/CD
- Плейбук полного развёртывания: `ansible/deploy_full.yml`
- GitHub Actions: `.github/workflows/ci.yml`
Пайплайн запускается при push в ветку main: проверяет синтаксис и делает сухой прогон.

### Управление секретами
- Пароль пользователя `deploy` хранится в виде хэша SHA-512 непосредственно в плейбуке `ansible/task4_create_user.yml`.
- В реальной среде пароли и другие секретные данные должны быть вынесены в зашифрованные файлы `ansible-vault` и не публиковаться в открытом доступе.
- Принцип работы с секретами продемонстрирован: хэш генерируется командой `python3 -c "import crypt; ..."`, затем вставляется в плейбук.
